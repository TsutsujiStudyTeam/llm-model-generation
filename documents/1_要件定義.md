# プロジェクト定義：LLM学習・推論お試しセット

## 1. プロジェクト概要

Llama 系（Unsloth 4bit）をベースとした LLM のファインチューニング（学習）から、成果物のデプロイ、および複数モデルを切り替えて試せる推論環境を「完全無料」かつ「エンジニアフレンドリー」に構築する。

## 2. コア要件

### A. 学習環境 (Training)

- **基盤**: Google Colab（無料版 T4 GPU を想定）
- **インターフェース**: VS Code の Google Colab 拡張機能の利用は任意。ブラウザ上の Colab でも可。
- **ライブラリ**: `Unsloth`（高速化・メモリ節約）
- **成果物**: 学習済み LoRA アダプタを Hugging Face Hub にアップロード（`push_to_hub`）
- **操作の最小化（必須）**:
  - 学習処理は **`training/finetune_script.py`** に集約し、**対話入力（`input` / `getpass` / `notebook_login`）を行わない**。
  - Hugging Face 認証は環境変数 **`HF_TOKEN`**（Google Colab では **Secrets** の `HF_TOKEN`）のみとする。
  - アップロード先リポジトリは **`training/params.yaml` の `hf_lora_repo`** で定義する（必要なら環境変数 **`HF_LORA_REPO`** で上書き可）。
  - ノートブック **`training/finetune.ipynb`** は、依存導入・リポジトリ clone・スクリプト実行までを **「すべてのセルを実行」** で完走できる構成とする。

### B. 推論環境 (Inference)

- **基盤**: Hugging Face Spaces（SDK: Gradio）
- **リソース**: 無料版 CPU Basic を想定
- **機能**:
  - ブラウザから操作可能なチャット UI
  - **動的モデル切り替え**: Hugging Face Hub 上の LoRA アダプタを選択してロードし推論できること

### C. 管理・運用

- **コード管理**: GitHub（コード、ノートブック、設定ファイル）
- **成果物管理**: Hugging Face Hub（モデル等）
- **秘匿情報**: トークンは `.env`（ローカル）または **Colab / Spaces の Secret** に保存し、リポジトリに直書きしない

## 3. 技術スタック

- **Base Model**: `training/params.yaml` の `hf_model_repo`（例: Unsloth 4bit Alpaca ベース）
- **Language**: Python 3.10+
- **Main Libraries**: `unsloth`, `gradio`, `transformers`, `peft`, `trl`, `huggingface_hub`
- **Data Format**: JSONL（`instruction`, `input`, `output`）
