# 運用設計書

本ドキュメントは、「LLM学習・推論お試しセット」の環境構築から利用までの手順を定義します。

## 1. 概要

運用は **学習フェーズ**（Colab）と **推論フェーズ**（Spaces 等）に分かれます。学習は **対話入力なし**（`HF_TOKEN` と `params.yaml`）で完走する前提です。

## 2. 学習手順（ファインチューニング）

### 2.1. 前提条件

- Google アカウント（Colab）
- Hugging Face アカウントと **Write** トークン
- Git が使えるローカル環境（`params.yaml` の編集・push 用）
- （任意）VS Code + Google Colab 拡張

### 2.2. 事前準備（GitHub 上のコード）

1. **Hugging Face で空のモデルリポジトリ**を作成する（例: `username/my-lora-adapter`）。
2. ローカルで `training/params.yaml` の **`hf_lora_repo`** をそのリポジトリ名に変更する。
3. `data/dataset.jsonl` を必要に応じて編集する。
4. 変更を `git commit` / `git push` する。

### 2.3. Colab Secrets（初回のみ）

1. Colab でノートブックを開く。
2. 左メニュー **🔑 シークレット** を開く。
3. 名前 **`HF_TOKEN`**、値に Hugging Face の Write トークンを登録する。
4. プライベート GitHub リポジトリを clone する場合のみ、**`GITHUB_TOKEN`**（`repo` スコープ）を追加する。

### 2.4. 学習の実行

1. `training/finetune.ipynb` を開く（README の「Open in Colab」バッジからでも可）。
2. **ランタイム → ランタイムのタイプを変更** で **GPU（T4 等）** を選択する。
3. **ランタイム → すべてのセルを実行**する。

ノートブックは pip インストール、clone、`python training/finetune_script.py` までを順に実行する。セル途中でトークン入力を求められない。

### 2.5. 成果物の確認

- Hugging Face の `hf_lora_repo` にファイルがアップロードされていることを確認する。
- 必要に応じてノートブック末尾の任意セルで Colab 上スモークテストを行う。

### 2.6. アップロード先を一時的に変えたい場合

- `params.yaml` を書き換えずに試す場合、Colab 上で学習セルの直前に環境変数を設定する（例: `%env HF_LORA_REPO=username/other-repo`）。詳細は README を参照。

## 3. 推論環境の構築と利用

### 3.1. Hugging Face Space

1. [Spaces](https://huggingface.co/spaces/new) で Gradio・CPU Basic の Space を作成する。
2. GitHub リポジトリと連携するか、`inference/` の内容を Space リポジトリに配置する（Space のルート構成に合わせる）。
3. **Settings → Secrets** に `HF_TOKEN` を設定する（プライベートモデル読み込み時など）。

### 3.2. ローカル推論（オプション）

- **仮想環境を作成し、その中で** `pip install -r inference/requirements.txt` を実行する。システムの Python へ直接インストールしない。
- `HF_TOKEN` を設定し、`python inference/app.py` で起動する。

## 4. 開発時のテスト（ローカル）

コードや設定の破綻を防ぐため、仮想環境で次を実行できる。

```text
pip install -r requirements-dev.txt
pytest tests/ -v
```

GPU や Unsloth は不要。CI や手元の事前チェックに利用する。

## 5. トラブルシューティング（運用）

| 現象 | 確認すること |
|------|----------------|
| `HF_TOKEN is not set` | Colab Secrets の名前が `HF_TOKEN` か。スペルミスがないか。 |
| `hf_lora_repo` エラー | `params.yaml` を自分の Hub 名に更新したか。 |
| clone 失敗 | プライベート repo なら `GITHUB_TOKEN` を Secrets に入れたか。 |
| GPU が付かない | ランタイムが GPU か。無料枠の都合で付かない場合がある。 |
