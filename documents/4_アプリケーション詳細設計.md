# アプリケーション詳細設計書

本ドキュメントは、「LLM学習・推論お試しセット」における各アプリケーションの設計を定義します。

## 1. 学習アプリケーション

学習は **Google Colab 上のノートブック**（`training/finetune.ipynb`）と **Python スクリプト**（`training/finetune_script.py`）の組み合わせで行います。実際の学習ロジックは **`finetune_script.py` に集約**します。

### 1.1. `training/finetune.ipynb`（Colab）

- **役割**: 依存パッケージのインストール、GitHub からの clone、環境変数の設定（Colab Secrets から `HF_TOKEN` / `HF_LORA_REPO` / 任意の `HF_MODEL_REPO`、任意の `GITHUB_REPO_OWNER`・`GITHUB_REPO_NAME`）、**毎回の学習データ指定**（セクション 2.5・`.jsonl` のみアップロード／ファイル選択）、`finetune_script.py` の起動、学習後の **対話式推論**（セクション 4・`input()`）。
- **対話なし（学習）**: 学習スクリプトは `notebook_login()` や学習用 `input()` を使わない。推論セルは対話用 `input()` を使用する。
- **clone**: 公開リポジトリは HTTPS でトークンなし clone。プライベートの場合のみ Secret `GITHUB_TOKEN` を使用。
- **任意セル**: 学習後に環境変数または `params.yaml` で解決したベースモデルと Hub の LoRA を読み込み、簡易推論するコードを含められる（マージ済みローカルフォルダの単体 `from_pretrained` は環境によって tokenizer エラーになり得るため使わない）。

### 1.2. `training/finetune_script.py`

- **認証**: 起動時に `HF_TOKEN` を必須とし、`huggingface_hub.login(token=...)` を実行する。
- **パラメータ**: `training/params.yaml` を読み込む。ベースモデルは `HF_MODEL_REPO` があれば優先、なければ `hf_model_repo`。LoRA のアップロード先は `HF_LORA_REPO` があれば優先、なければ `hf_lora_repo`。プレースホルダ `YOUR_USERNAME` を含む `hf_lora_repo` のまま（かつ環境変数未設定）の場合はエラーとする。
- **データ**: 優先順は環境変数（または Colab Secret）`TRAINING_DATASET_PATH` → `TRAINING_DATASET_URL`（`https` でダウンロードし `training/.dataset_cache/` にキャッシュ）→ `params.yaml` の `dataset_jsonl`（既定 `data/dataset.jsonl`）。いずれも Alpaca 形式（`instruction` / `input` / `output`）の JSONL。
- **学習**: `Unsloth` の `FastLanguageModel` と `trl.SFTTrainer` により LoRA 学習を実行。
- **保存**: `training/lora_adapter_merged` にマージ済みアダプタを保存。
- **アップロード**: `model.push_to_hub` / `tokenizer.push_to_hub` に `token=HF_TOKEN` を明示する。

### 1.3. 設定ファイル (`training/params.yaml`)

学習の挙動と Hub 上のモデル ID の**既定値**を定義する（Colab ではシークレット `HF_MODEL_REPO` / `HF_LORA_REPO` があればこちらより優先）。

```yaml
hf_model_repo: "unsloth/Llama-3.2-3B-Instruct-bnb-4bit"
hf_lora_repo: "YOUR_USERNAME/YOUR_LORA_REPO"
dataset_jsonl: "data/dataset.jsonl"
# 以下、LoRA 次元、学習ステップ等
```

## 2. 推論アプリケーション (`inference/app.py`)

Hugging Face Spaces 上で動作する Gradio アプリを想定したコードです。

### 2.1. 処理の概要

1. 起動時またはリクエスト時に、ベースモデル（`BASE_MODEL_REPO`）を 4bit 等でロードする。
2. ドロップダウンで指定された LoRA アダプタを Hub から読み込む（`HF_TOKEN` でプライベートリポジトリにアクセス可能にする）。
3. `gr.ChatInterface` によりチャット UI を提供する。

### 2.2. 依存ライブラリ (`inference/requirements.txt`)

Spaces / ローカルではこのファイルを用いて依存をインストールする。CUDA バージョンは実行環境に合わせて調整する場合がある。

## 3. データフォーマット (`data/dataset.jsonl`)

JSON Lines 形式。各行に以下のキーを持つ JSON オブジェクトを置く。

- `instruction`: 指示
- `input`: 追加文脈（空でも可）
- `output`: 期待する応答

例は README および本リポジトリの `data/dataset.jsonl` を参照。

## 4. ローカル検証（GPU 不要）

`tests/test_project_layout.py` および `pytest` により、`params.yaml` の存在・必須キー、`dataset.jsonl` の存在、`finetune_script.py` の構文を検証する。実行には `requirements-dev.txt` を仮想環境にインストールする。
