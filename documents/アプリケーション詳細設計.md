# アプリケーション詳細設計書

本ドキュメントは、「LLM学習・推論お試しセット」プロジェクトにおける各アプリケーションの具体的な設計を定義します。

## 1. 学習アプリケーション (`training/finetune.ipynb`)

Google Colab 上で実行される、LLMのファインチューニング用Jupyter Notebookです。

### 1.1. 処理フロー

1.  **環境設定**
    - `pip` を使用し、`unsloth`, `transformers`, `peft`, `trl`, `huggingface_hub` 等の必要ライブラリをインストールします。
    - Google Colab の `secrets` 機能から Hugging Face トークンを読み込み、`huggingface_hub` にログインします。

2.  **パラメータ読み込み**
    - `params.yaml` から学習に関するパラメータ（モデル名, LoRA設定, 学習ハイパーパラメータ等）を読み込みます。

3.  **モデルとトークナイザの準備**
    - `unsloth` の `FastLanguageModel.from_pretrained` を使用し、4bit量子化されたベースモデル (`Llama-3.2-3B-Instruct`) とトークナイザをロードします。この際、`Unsloth` による高速化パッチが適用されます。

4.  **データセットの準備**
    - `data/dataset.jsonl` を `datasets` ライブラリで読み込みます。
    - 各レコードをモデルが学習しやすい形式（例：チャットテンプレート形式）にフォーマットする前処理関数を定義し、適用します。

5.  **LoRA (PEFT) 設定**
    - `peft` の `LoraConfig` に基づき、LoRAの設定（`r`, `lora_alpha`, `target_modules`, `task_type` 等）を定義します。`Unsloth` は最適な `target_modules` を自動で検出する機能を提供するため、これを活用します。

6.  **学習の実行**
    - `trl` の `SFTTrainer` を使用して学習タスクを定義します。
    - `SFTTrainer` には、モデル、トークナイザ、学習データセット、LoRA設定、および学習ハイパーパラメータ（`per_device_train_batch_size`, `gradient_accumulation_steps`, `warmup_steps`, `max_steps` 等）を渡します。
    - `trainer.train()` を呼び出し、ファインチューニングを開始します。

7.  **成果物の保存とアップロード**
    - 学習完了後、`model.push_to_hub` メソッドを実行し、学習済みLoRAアダプタをHugging Face Hubの指定リポジトリに直接アップロードします。

### 1.2. 設定ファイル (`training/params.yaml`)

学習の挙動を制御するためのパラメータを定義します。

```yaml
# Hugging Face Settings
hf_model_repo: "google/Llama-3.2-3B-Instruct" # ベースモデル
hf_lora_repo: "YOUR_HF_USERNAME/Llama-3.2-3B-Instruct-lora-test" # LoRAアダプタの保存先

# LoRA Settings
lora_r: 16
lora_alpha: 32
lora_dropout: 0.05
lora_target_modules: "auto" # Unslothによる自動検出

# Training Settings
max_seq_length: 2048
per_device_train_batch_size: 2
gradient_accumulation_steps: 4
warmup_steps: 10
max_steps: 60
learning_rate: 0.0002
fp16: true
logging_steps: 1
```

## 2. 推論アプリケーション (`inference/app.py`)

Hugging Face Spaces 上で動作する、Gradioベースのチャットアプリケーションです。

### 2.1. 処理フロー

1.  **初期化**
    - 起動時に、`unsloth` を使用してベースモデル (`Llama-3.2-3B-Instruct`) を4bit量子化でロードします。このモデルはアダプタが適用されていない、素の状態です。
    - Hugging Face Hub API (`huggingface_hub.list_models`) を使用し、指定したユーザー or organization 配下のアダプタ（モデル）一覧を取得し、プルダウンメニューの選択肢を生成します。

2.  **Gradio UIの構築**
    - `gr.Blocks` を使用してUIのレイアウトを定義します。
    - `gr.Dropdown` を配置し、取得したLoRAアダプタの一覧を表示します。
    - `gr.ChatInterface` を配置し、チャット用のUIを提供します。

3.  **動的モデル切り替え**
    - `gr.Dropdown` の値が変更されたことをトリガー (`.change()`) として、以下の関数を実行します。
        - **関数 `load_adapter(adapter_name)`**:
            1. 現在のモデルに適用されているアダプタを無効化します (`model.disable_adapter()`)。
            2. 選択された `adapter_name` に対応するLoRAアダプタを `model.load_adapter(adapter_name)` でロードします。
            3. 成功メッセージをUIに返します。

4.  **推論処理**
    - `gr.ChatInterface` の入力（ユーザーメッセージ）を処理する関数 (`predict`) を定義します。
        - **関数 `predict(message, history)`**:
            1.  `history` を含むプロンプトを、モデルのチャットテンプレートに従って整形します。
            2.  整形したプロンプトをトークナイズし、モデルに入力します。
            3.  `model.generate` を実行し、ストリーミング形式で応答を生成します。
            4.  生成されたテキストを随時 `yield` で返し、`gr.ChatInterface` にタイプライターのように表示させます。

## 3. データフォーマット (`data/dataset.jsonl`)

学習に使用するデータセットは、JSON Lines形式で提供されます。各行は1つのJSONオブジェクトであり、以下のキーを持ちます。

- `instruction`: モデルへの指示・質問
- `input`: 指示に対する文脈や追加情報（任意）
- `output`: モデルが生成すべき理想的な応答

**例:**
```json
{"instruction": "3つのリンゴと2つのオレンジを合わせると、果物は全部でいくつになりますか？", "input": "", "output": "3つのリンゴと2つのオレンジを合わせると、合計で5つの果物になります。"}
{"instruction": "与えられた文章の要点を3つ挙げてください。", "input": "人工知能（AI）は、現代社会の多くの側面に影響を与えています。特に、機械学習という分野は、データからパターンを自動的に学習する能力を持ち、画像認識、自然言語処理、医療診断など、多岐にわたる応用例があります。しかし、AIの発展は倫理的な課題も提起しており、公平性、透明性、プライバシーの確保が今後の重要なテーマとなります。", "output": "1. AI、特に機械学習は、画像認識や自然言語処理など多方面で応用されている。\n2. 機械学習は、データから自動でパターンを学習する技術である。\n3. AIの発展に伴い、公平性やプライバシーといった倫理的な課題への対応が重要になっている。"}
```
